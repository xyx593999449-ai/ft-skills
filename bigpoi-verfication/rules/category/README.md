# 分类正确性核实规则

本目录包含用于验证POI分类是否与实体属性匹配的规则。

## 规则清单

### 1. 完全分类匹配规则

**文件**：`exact_category_match.yaml`

**功能**：输入分类与证据分类完全一致

**应用条件**：
- 输入分类编码 == 证据分类编码

**输出**：
- result: pass
- confidence: 1.0

### 2. 分类相同概念规则

**文件**：`category_synonym.yaml`

**功能**：不同的分类编码但代表相同的概念

**应用条件**：
- 两个分类属于同一分类体系的同一层级
- 语义相同但编码不同

**输出**：
- result: pass
- confidence: 0.95

### 3. 分类相关性规则

**文件**：`category_relevant.yaml`

**功能**：分类存在关联但不完全相同

**应用条件**：
- 两个分类存在层级关系（如医院与综合医院）
- 相似度 >= 0.70

**输出**：
- result: uncertain
- confidence: 0.75

### 4. 分类不匹配规则

**文件**：`category_mismatch.yaml`

**功能**：分类明显不匹配或不相关

**应用条件**：
- 分类无任何关联
- 相似度 < 0.70

**输出**：
- result: fail
- confidence: 0.3

## 分类编码体系

支持以下分类编码体系：

- **POI标准编码**：6位数字编码（如090100表示综合医院）
- **通用编码**：中文分类名称
- **第三方编码**：高德、百度等图商的分类编码

## 分类映射表

建议维护一份分类映射表，用于在不同分类体系之间进行转换。

## 规则应用顺序

1. 完全匹配检查（优先级1）
2. 同义词检查（优先级2）
3. 相关性检查（优先级3）
4. 不匹配检查（优先级4）
