# 名称准确性核实规则

本目录包含用于验证POI名称是否与官方命名一致的规则。

## 规则清单

### 1. 完全匹配规则

**文件**：`exact_match.yaml`

**功能**：检查输入名称与证据名称的完全匹配

**应用条件**：
- 输入名称 == 证据名称

**输出**：
- result: pass
- confidence: 1.0

### 2. 高相似度匹配规则

**文件**：`high_similarity.yaml`

**功能**：通过相似度算法判断名称是否基本一致

**应用条件**：
- 名称相似度 >= 0.85

**输出**：
- result: pass
- confidence: 0.90

### 3. 中等相似度不确定规则

**文件**：`medium_similarity.yaml`

**功能**：名称相似度在可接受范围但需要进一步确认

**应用条件**：
- 0.70 <= 名称相似度 < 0.85

**输出**：
- result: uncertain
- confidence: 0.75

### 4. 低相似度失败规则

**文件**：`low_similarity.yaml`

**功能**：名称差异过大，判定失败或降级

**应用条件**：
- 名称相似度 < 0.70

**输出**：
- result: fail
- confidence: 0.4

## 相似度计算方法

使用Levenshtein距离或SequenceMatcher算法计算：

```
相似度 = 匹配字符数 / max(len(名称1), len(名称2))
```

## 规则应用顺序

1. 完全匹配检查（优先级1）
2. 高相似度检查（优先级2）
3. 中等相似度检查（优先级3）
4. 低相似度检查（优先级4）
